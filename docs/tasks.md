# Tasks: 実装タスク分解

各タスクは独立してレビュー・マージできる粒度を意識している。上から順に進めることを推奨。Codexに1タスクずつ渡すことを想定。

## Phase 0: 基盤

- [ ] T-01: Nuxt 4(TypeScript)プロジェクトの雛形作成、Tailwind CSSモジュール導入
- [ ] T-02: Prisma + PostgreSQLのセットアップ、`design.md`のスキーマでマイグレーション作成
- [ ] T-03: Docker Compose(app + postgres)でローカル起動できる状態にする
- [ ] T-04: 初期管理者ユーザーをseedスクリプトで作成できるようにする

## Phase 1: 認証・管理画面の骨格

- [ ] T-05: nuxt-auth-utilsによるセッションベースのログイン機能(Credentials方式)
- [ ] T-06: 管理画面のレイアウト(`app/layouts/admin.vue`、サイドナビ、ダッシュボードの空実装)
- [ ] T-07: ダッシュボード画面:マップ一覧の表示、新規マップ作成ボタン

## Phase 2: マップ・フロア管理

- [ ] T-08: マップ設定画面:マップ名の登録、マップの新規作成API
- [ ] T-09: イラスト画像アップロード機能(API Route経由、ローカル保存でまず動かす)
- [ ] T-10: フロア(MapFloor)のCRUD:追加・並び替え・削除
- [ ] T-11: フロアの矩形ジオリファレンス設定UI(住所検索またはパン・ズームで対象地域へ移動し、軸沿い矩形をドラッグ移動・四隅ハンドルでリサイズ。矩形内にイラストを半透明表示し、左上・右下の2点を保存してMapLibreの`image`ソース用4隅を算出)

## Phase 3: スポット管理(管理側)

- [ ] T-15: 住所入力→Nominatim経由の自動ジオコーディング
- [ ] T-16: ピン配置エディタ:MapLibre地図上でクリックしてピンを仮配置(緯度経度を取得)
- [ ] T-17: 既存ピン(Marker)のドラッグによる位置調整
- [ ] T-18: ピンデザイン選択(プリセットアイコン一覧、カラーピッカー、カスタム画像アップロード)
- [ ] T-19: スポットの公開/非公開切り替え、プレビュー機能

## Phase 4: 地図ビューア(MapLibre GL JS, コアコンポーネント)

- [ ] T-21: フロアのイラストを`image`ソースとして追加し、緯度経度に対応づけて表示
- [ ] T-22: `pitch`/`bearing`/`zoom`の制約設定(`maxPitch`等。design.md 4.2節を参照)
