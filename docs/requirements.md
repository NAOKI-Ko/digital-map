# Requirements: デジタルマップ作成プラットフォーム(MVP)

## 1. プロダクトの目的

非エンジニアの団体(商店街振興組合、温泉旅館組合、地下街管理会社など)が、自分たちでデジタルマップを作成・運用できるプラットフォーム。プラチナマップに近い立ち位置を狙う。

## 2. MVPスコープの原則

- 対象機能は「マップの作成・編集・閲覧」のみ
- 以下は明示的にスコープ外(次フェーズ):
  - CSV一括インポート
  - 店舗オーナーによる自店舗編集・承認フロー
  - モデルコース/ルート機能
  - お知らせ・イベント機能
  - 多言語対応
  - クーポン・スタンプラリー機能
  - マップのテンプレート/ブランディング設定(ロゴ・配色)
  - カテゴリ単位のデフォルトピンデザイン一括設定

## 3. ロールとユーザーストーリー

### 3.1 管理者(組合・協会の担当者)

- US-01: 管理者として、イラストマップ画像をアップロードして、マップの土台を作りたい
- US-02: 管理者として、地下街のような複数フロアがある場合、フロアごとに別のイラストを登録したい
- US-03: 管理者として、イラスト上と実際の地図上でそれぞれ2箇所の目印をクリックするだけで、
  イラストを実世界の位置に対応づけたい(4隅を個別に合わせるような難しい操作はしたくない)
- US-04: 管理者として、既存ピンをドラッグして位置を微調整したい
- US-05: 管理者として、スポットの基本情報(店名・カテゴリ・説明文・写真・営業時間・定休日・電話番号)を登録・編集したい
- US-06: 管理者として、ピンの見た目(プリセットアイコン/カラー/カスタム画像)を自由にカスタマイズしたい
- US-07: 管理者として、スポットを下書き/公開の状態で管理したい
- US-08: 管理者として、マップ全体を公開/非公開に切り替え、公開URLとQRコードを取得したい
- US-09: 管理者として、登録済みスポットを一覧で確認・検索・絞り込みしたい

### 3.2 一般閲覧者(観光客・地域住民)

- US-10: 閲覧者として、マップをドラッグして傾け・回転させながら全体を眺めたい
- US-11: 閲覧者として、ピンをタップしてスポットの詳細情報を見たい
- US-12: 閲覧者として、カテゴリで絞り込んでスポットを探したい
- US-13: 閲覧者として、複数フロアがある場合、フロアを切り替えて見たい
- US-14: 閲覧者として、屋外エリアでは自分の現在地がマップ上のだいたいどこかを確認したい(地下街では対象外)

## 4. 機能要件(MVP)

| ID | 要件 | 優先度 |
|---|---|---|
| FR-01 | イラスト画像(png/jpg)のアップロード | Must |
| FR-02 | 2点合わせによるジオリファレンス(イラスト上・実地図上で各2箇所の目印をクリックし、位置・回転・縮尺を自動計算) | Must |
| FR-03 | フロア(イラスト画像単位)の追加・並び替え・削除 | Must |
| FR-04 | 住所入力→自動ジオコーディングによるピン配置、および基準点検索の補助 | Must |
| FR-05 | イラスト上のクリック/ドラッグによるピン位置の手動設定・調整 | Must |
| FR-06 | スポット情報の登録編集フォーム(店名/カテゴリ/説明文/写真/営業時間/定休日/電話番号) | Must |
| FR-07 | ピンデザインのカスタマイズ(プリセットアイコン/カラー/カスタム画像) | Must |
| FR-08 | スポット一覧・検索・絞り込み(管理側) | Must |
| FR-09 | マップの公開/非公開切り替え、公開URL・QRコード発行 | Must |
| FR-10 | 公開マップの傾け・回転・ズームができる擬似3Dビューア | Must |
| FR-11 | ピンタップによるスポット詳細のオーバーレイ表示 | Must |
| FR-12 | カテゴリ絞り込み(閲覧側) | Must |
| FR-13 | フロア切り替え(閲覧側、複数フロアがある場合のみ表示) | Must |
| FR-14 | 現在地取得・表示(屋外のジオリファレンス済みフロアのみ対応。地下街は対象外) | Should |
| FR-15 | 管理者ログイン・認証 | Must |

## 5. 非機能要件

- NFR-01: 非エンジニアの管理者が迷わず操作できるUI(専門用語を避ける、入力ステップを最小限にする)
- NFR-02: モバイル端末での閲覧を最優先(観光客はスマホ利用が前提)。管理画面はPC優先だがレスポンシブ対応
- NFR-03: セルフホスト可能なこと(Docker Composeで起動できる)を将来的なOSS配布に備えて意識する
- NFR-04: 現在地の精度は、選んだ2つの基準点の位置関係やイラストのデフォルメ度合いに依存することを利用者に誤解させない(過度に正確な現在地表示を演出しない)

## 6. 対象外(スコープ外)の明記

- 現在地取得は地下街では非対応(GPSが届かないため、`is_outdoor = false`のフロアでは`GeolocateControl`を追加しない)
- 現在地取得は、選んだ基準点の位置関係やイラストのデフォルメ度合いによって精度が変わる。正確なナビゲーションとしては保証しない
